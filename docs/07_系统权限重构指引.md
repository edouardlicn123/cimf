# 系统权限模块重构指引

本文档描述系统权限模块的重构方案，包括权限细化、卡片化显示等内容。

---

## 一、当前问题

当前系统权限配置存在以下问题：

1. **权限粒度粗**：系统设置、权限管理、人员管理都只有一个权限项，无法精细控制
2. **显示样式单一**：系统权限使用表格样式，与节点类型权限的卡片样式不一致
3. **缺乏图标标识**：系统权限缺少直观的图标标识

---

## 二、重构方案

### 2.1 权限细化

将原有的3个系统权限细化为9个权限，按模块分组：

| 模块 | 权限键 | 说明 |
|------|--------|------|
| 系统设置 | `system.settings.view` | 系统设置 - 查看 |
| 系统设置 | `system.settings.modify` | 系统设置 - 修改 |
| 权限管理 | `permissions.view` | 权限管理 - 查看 |
| 权限管理 | `permissions.modify` | 权限管理 - 修改 |
| 人员管理 | `user.create` | 人员管理 - 创建 |
| 人员管理 | `user.read` | 人员管理 - 查看 |
| 人员管理 | `user.update` | 人员管理 - 修改 |
| 人员管理 | `user.delete` | 人员管理 - 删除 |

### 2.2 图标分配

每个系统权限模块分配一个 Bootstrap Icons 图标：

| 模块 | 图标 | 说明 |
|------|------|------|
| 系统设置 | `bi-gear` | 系统设置模块 |
| 权限管理 | `bi-shield-lock` | 权限管理模块 |
| 人员管理 | `bi-people` | 人员管理模块 |

### 2.3 显示样式统一

系统权限卡片样式与节点类型权限卡片保持一致：

```html
<!-- 系统权限卡片 -->
<div class="card mb-3">
  <div class="card-header bg-light">
    <i class="bi {icon} me-2"></i>{模块名称}
  </div>
  <div class="card-body p-0">
    <table class="table table-bordered table-hover mb-0">
      <tbody>
        {% for perm_key, perm_label in data.permissions %}
        <tr>
          <td style="min-width: 200px;"><label class="form-check-label">{{ perm_label }}</label></td>
          <td class="text-center" style="width: 120px;">
            <input type="checkbox" class="form-check-input" name="permissions_leader" value="{{ perm_key }}" {% if perm_key in role_permissions['leader'] %}checked{% endif %}>
          </td>
          <td class="text-center" style="width: 120px;">
            <input type="checkbox" class="form-check-input" name="permissions_employee" value="{{ perm_key }}" {% if perm_key in role_permissions['employee'] %}checked{% endif %}>
          </td>
        </tr>
        {% endfor %}
      </tbody>
    </table>
  </div>
</div>
```

卡片结构：
- **卡片标题**：显示模块名称和图标（如 `bi-gear` + "系统设置"）
- **卡片内容**：表格形式，包含权限名称、leader列、employee列
- **样式**：与节点类型权限卡片保持一致（`bg-light` 背景、两列布局）

## 三、代码修改

### 3.1 PermissionService 修改

#### 3.1.1 更新 PERMISSIONS 常量

文件：`app/services/core/permission_service.py`

```python
# 可用权限列表（可扩展）
PERMISSIONS = [
    # 系统设置 - 细化为查看和修改
    ('system.settings.view', '系统设置 - 查看'),
    ('system.settings.modify', '系统设置 - 修改'),
    
    # 权限管理 - 细化为查看和修改
    ('permissions.view', '权限管理 - 查看'),
    ('permissions.modify', '权限管理 - 修改'),
    
    # 人员管理 - 细化为 CRUD
    ('user.create', '人员管理 - 创建'),
    ('user.read', '人员管理 - 查看'),
    ('user.update', '人员管理 - 修改'),
    ('user.delete', '人员管理 - 删除'),
]
```

#### 3.1.2 新增 get_system_permissions 方法

```python
@staticmethod
def get_system_permissions() -> Dict[str, Dict]:
    """获取系统权限，按模块分组"""
    return {
        'system_settings': {
            'name': '系统设置',
            'icon': 'bi-gear',
            'permissions': [
                ('system.settings.view', '系统设置 - 查看'),
                ('system.settings.modify', '系统设置 - 修改'),
            ]
        },
        'permissions': {
            'name': '权限管理',
            'icon': 'bi-shield-lock',
            'permissions': [
                ('permissions.view', '权限管理 - 查看'),
                ('permissions.modify', '权限管理 - 修改'),
            ]
        },
        'user': {
            'name': '人员管理',
            'icon': 'bi-people',
            'permissions': [
                ('user.create', '人员管理 - 创建'),
                ('user.read', '人员管理 - 查看'),
                ('user.update', '人员管理 - 修改'),
                ('user.delete', '人员管理 - 删除'),
            ]
        },
    }
```

#### 3.1.3 更新 get_all_permissions 方法

```python
@staticmethod
def get_all_permissions() -> List[tuple]:
    """获取所有可用权限列表"""
    return PERMISSIONS
```

> **说明**：`get_all_permissions()` 返回扁平列表用于兼容性，`get_system_permissions()` 返回分组字典用于模板渲染。

### 3.2 权限管理路由修改

文件：`app/modules/core/admin/routes.py`

修改 `permissions_edit` 函数中 `all_permissions` 的构建逻辑：

```python
# 获取系统权限（分组）
system_permissions = PermissionService.get_system_permissions()
# 获取Node权限（分组）
node_permissions = PermissionService.get_node_permissions()

# 构建扁平权限列表用于兼容性
all_perms_list = []
for data in system_permissions.values():
    all_perms_list.extend(data['permissions'])
for data in node_permissions.values():
    all_perms_list.extend(data['permissions'])

return render_template(
    'core/admin/system_permissions.html',
    form=form,
    role_name_form=role_name_form,
    system_permissions=system_permissions,
    node_permissions=node_permissions,
    all_permissions=all_perms_list,
    role_permissions=role_permissions,
    role_labels=role_labels,
    active_section='permissions'
)
```
